
ELMS Instructional Templates API
--------------------------------

This API is fairly straight forward.  You can use it to get new book outlines automatically populated on the creation of new course versions.  Some samples are provided in the elms_instructional_templates_elms_instructional_template function.

Here is the basic structure:
$outline['YOUR_TEMPLATE_MACHINE_NAME'] = array(
     'title' => 'YOUR TEMPLATE NAME',
     0 => array(
       'type' => ,
       'title' => ,
       'link' => ,
       'parent' => 
     ));

--Acceptable values for type is link, page, folder, or duplicate
--'link' is only used if type is link
--@i can be used in the title of something that's going to be duplicated as to where the incemental value should be placed.  This is not required.
--parent is a reference to the array member preceeding it, here's a visual example:

0 -- Thing1 (no parent listed at base level)
1 ---- Stuff2 (parent is 0)
2 ------ Herethere3 (parent is 1)
3 ---- MoreStuff (parent is 0)
4 -- Whothere (no parent listed at base level)

--in the case of duplicate, the parent value is what you want to duplicate.
--nested duplication is untested and not recommended

/**
 * Implementation of hook_elms_instructional_template().
 */
function YOURMODULENAME_elms_instructional_template() {
  $outline['YOUR_TEMPLATE_MACHINE_NAME'] = array(
    'title' => 'YOUR TEMPLATE NAME',
    0 => array(
      'type' => 'link',
      'title' => 'Homepage',
      'link' => '<front>',
    ),
    1 => array(
      'type' => 'folder',
      'title' => t('Course Content'),
    ),
    2 => array(
      'type' => 'page',
      'title' => t('Lesson @i'),
      'parent' => 1,
    ),
    3 => array(
      'type' => 'duplicate',
      'parent' => 2,
    ),
  );
  return $outline;
}
